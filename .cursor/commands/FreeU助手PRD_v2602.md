## 一、产品概述

### 1.1 产品定位

**Personal Proactive Agent（个人主动服务代理）** —— 一款能够7×24小时通过**音频+PC屏幕**多模态感知用户生活、主动提供智能建议的AI助手。

核心理念：将用户生活中、电脑中、与他人沟通中的所有相关信息组织在一起，让AI真正成为你的"军师"。

### 1.2 核心价值主张

- **持续感知**：用户无需反复交代背景信息，AI已掌握90%以上的生活上下文
- **主动服务**：不需要用户主动发起对话，AI能在合适的时机自动介入
- **深度个性化**：基于用户长期积累的上下文数据，提供真正"懂你"的服务

## 三、核心功能需求

### 3.1 感知模块（数据采集）

#### 3.1.1 音频采集

- **硬件**：独立录音设备（录音豆/耳机），可7×24小时携带，电脑麦克风（开发阶段先用这个）
- **格式**：实时流式传输
- **处理**：语音识别 → 说话人区分（音色识别） → 文本化

#### 3.1.2 屏幕采集（LiveTrace）

- **截图**：定时截屏
- **OCR**：文字提取
- **GUI解析**：利用已知的APP布局先验（微信、飞书等）
- **活动聚合**：多个事件 → 生成一个Activity（summary + 进展 + 下一步）

### 3.2 个人知识库模块

- **数据存储**：本地优先
- **数据组织**：压缩
- **数据更新**：实时增量更新
- **隐私保护**：优先本地处理
- **数据来源**：
  - 音频转录文本（携带说话人标签）
  - 屏幕截图OCR文本（携带来源APP标签）
  - 手动输入

### 3.3 主动意图识别模块（核心差异化）

#### 3.3.1 意图来源分类

主动服务的核心是**意图检测**，人的意图来源分为两类：

| 类型                 | 描述                                                 | 检测难度 | 优先级 |
| -------------------- | ---------------------------------------------------- | -------- | ------ |
| **社交传递型** | 别人通过某种媒介（屏幕消息、语音对话）把信息传递给你 | 较易     | P0     |
| **自发产生型** | 你脑子里自发产生的想法                               | 较难     | P2     |

**先做社交传递型**：这类意图非常清晰，如"导师问能不能今天下午开个会"，一定是通过某种媒介到达用户感官的。

#### 3.3.2 小模型初筛（When）

小模型负责实时、低成本地完成第一道筛选，判断是否需要触发大模型：

| 优先级 | 检测场景   | 检测方法 | 技术方案       |
| ------ | ---------- | -------- | -------------- |
| P0     | 收到疑问句 | 句式检测 | 小模型快速识别 |
| P0     | 收到祈使句 | 句式检测 | 小模型快速识别 |

### 3.4 规划与执行模块

小模型初筛通过后，由**大模型**接管，负责进一步的意图分析、信息检索、规划与执行：

```
小模型初筛通过 → 触发大模型
      ↓
┌─────────────────────────────────────────┐
│            大模型（强力）                 │
│                                          │
│  1. 进一步意图分析                        │
│     - 理解复杂意图、消歧                  │
│     - 判断用户真正需要什么                │
│                                          │
│  2. 内容检索（What）                      │
│     - 基于个人知识库的RAG检索             │
│     - 跨时间线信息关联                    │
│     - 联想：用户没想到的内容AI能想到       │
│                                          │
│  3. 规划与响应（How）                     │
│     - 建议回复（帮用户想回复话术）         │
│     - 尝试回答（先帮用户回答问题）         │
│     - 自动执行（指派Sub-Agent完成任务）    │
└─────────────────┬───────────────────────┘
                  ↓ 需要执行任务时
┌─────────────────────────────────────────┐
│           Sub-Agent调度                  │
│  任务分类 → 选择Agent → 执行 → 推送结果 │
└─────────────────────────────────────────┘
```

## 四、性能指标要求

### 4.1 Performance（效果）

效果指标对应主动服务的两个核心问题：

#### When — 意图检测准确率

| 指标           | 目标值                         | 说明                                   |
| -------------- | ------------------------------ | -------------------------------------- |
| 意图检测准确率 | ≥75%（MVP）→ ≥85%（成熟期） | 减少假阳性（打扰用户）和假阴性（漏检） |

#### How — 响应内容质量

| 指标           | 目标值 | 说明                             |
| -------------- | ------ | -------------------------------- |
| 响应内容准确率 | 待定   | 给出的建议/回复是否真正有用      |
| 信息检索召回率 | 待定   | 从个人知识库中找到相关信息的比例 |

### 4.2 延迟

| 指标           | 目标值  | 说明                           |
| -------------- | ------- | ------------------------------ |
| 端到端响应延迟 | ≤3秒   | 从检测到需求到给出响应的总时长 |
| 意图检测延迟   | ≤500ms | 小模型初筛的实时性要求         |

### 4.3 成本消耗

| 指标              | 目标值  | 说明                       |
| ----------------- | ------- | -------------------------- |
| 单次服务Token消耗 | 尽量低  | 保证商业可行性             |
| 设备续航          | ≥8小时 | 硬件录音设备               |
| 月均Token成本     | 待定    | 单用户月成本需低于订阅价格 |

### 4.4 感知覆盖

| 指标       | 目标值 | 说明                                     |
| ---------- | ------ | ---------------------------------------- |
| 数据覆盖率 | ≥90%  | 用户日常生活的信息覆盖，是一切效果的前提 |

## 五、开发路线（三步走）

### 5.1 第一步：搭Pipeline，验证Performance

先把音频+图像两条链路跑通，API先用最强模型，不考虑成本。

### 5.2 第二步：量化指标

建立指标体系（Performance、延迟、成本），构造数据集，找到堵点。

### 5.3 第三步：优化，验证商业可行性

小模型微调降本、大模型选型比价、先验利用减少Token，让商业模式跑通。
