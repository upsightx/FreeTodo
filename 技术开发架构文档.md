## 架构概述

### 架构设计原则

FreeTodo 项目遵循以下架构设计原则：

| 原则 | 描述 | 实践 |
|------|------|------|
| **前后端分离** | 前端和后端完全解耦 | 通过 RESTful API 通信 |
| **分层架构** | 职责清晰，层次分明 | Router → Service → Repository → Storage |
| **本地优先** | 数据存储在本地 | SQLite + ChromaDB 本地数据库 |
| **AI 原生** | AI 深度集成到核心流程 | LLM + RAG 检索增强 |
| **可扩展性** | 易于扩展新功能 | 模块化设计、插件式面板 |

### 各层职责

| 层级 | 职责 | 关键文件 |
|------|------|----------|

| **服务层** | 业务逻辑、编排多个 Repository 操作 | `services/*.py` |
| **仓储层** | 数据访问抽象、封装数据库查询 | `repositories/*.py` |
| **模式层** | Pydantic 数据模型（请求/响应） | `schemas/*.py` |

### 核心模块说明

#### 1. 路由模块 (routers/)

```python
# routers/todo.py 示例
from fastapi import APIRouter, Depends, Query
from lifetrace.services.todo_service import TodoService
from lifetrace.schemas.todo import TodoCreate, TodoResponse

router = APIRouter(prefix="/api/todos", tags=["todos"])

@router.get("/", response_model=list[TodoResponse])
async def list_todos(
    skip: int = Query(0, ge=0),
    limit: int = Query(50, ge=1, le=100),
    todo_service: TodoService = Depends(get_todo_service)
):
    """获取待办列表"""
    return await todo_service.list_todos(skip=skip, limit=limit)

@router.post("/", response_model=TodoResponse, status_code=201)
async def create_todo(
    todo: TodoCreate,
    todo_service: TodoService = Depends(get_todo_service)
):
    """创建待办"""
    return await todo_service.create_todo(todo)
```

#### 2. 服务模块 (services/)

```python
# services/todo_service.py 示例
class TodoService:
    """待办服务 - 业务逻辑层"""
    
    def __init__(self, todo_repository: TodoRepository):
        self.todo_repository = todo_repository
    
    async def create_todo(self, todo_data: TodoCreate) -> Todo:
        """创建待办（包含业务逻辑）"""
        # 业务验证
        if len(todo_data.title) > 200:
            raise ValueError("待办标题过长")
        
        # 处理父子关系
        if todo_data.parent_id:
            parent = await self.todo_repository.get_by_id(todo_data.parent_id)
            if not parent:
                raise ValueError("父任务不存在")
        
        return await self.todo_repository.create(todo_data)
```

// lib/generated/fetcher.ts - 自定义 Fetcher
export const customFetcher = async <T>(config: RequestConfig): Promise<T> => {
  const { url, method, data, params } = config
  
  // 1. 命名风格转换：camelCase → snake_case
  const transformedData = camelToSnake(data)
  
  // 2. 发送请求
  const response = await fetch(url, {
    method,
    body: JSON.stringify(transformedData),
    headers: { 'Content-Type': 'application/json' },
  })
  
  // 3. 响应转换：snake_case → camelCase
  const result = await response.json()
  return snakeToCamel(result) as T
}

class VectorDatabase:
    """ChromaDB 向量数据库封装"""
    
    def __init__(self):
        self.client = chromadb.PersistentClient(path="data/vector_db")
        self.collection = self.client.get_or_create_collection(
            name="lifetrace",
            embedding_function=SentenceTransformerEmbeddingFunction()
        )
    
    async def add_documents(self, documents: list[str], metadata: list[dict]):
        """添加文档到向量库"""
        embeddings = self.embedding_model.encode(documents)
        self.collection.add(
            documents=documents,
            embeddings=embeddings,
            metadatas=metadata,
            ids=[str(uuid.uuid4()) for _ in documents]
        )
    
    async def query(self, query: str, n_results: int = 5) -> list[dict]:
        """语义搜索"""
        results = self.collection.query(
            query_texts=[query],
            n_results=n_results
        )
        return results

import { app, BrowserWindow, Tray, Menu } from 'electron'
import { spawn } from 'child_process'

class Application {
  private mainWindow: BrowserWindow | null = null
  private tray: Tray | null = null
  private backendProcess: ChildProcess | null = null
  
  async init() {
    // 启动后端服务
    await this.startBackend()
    
    // 创建主窗口
    this.createMainWindow()
    
    // 创建系统托盘
    this.createTray()
  }
  
  private async startBackend() {
    // 在打包版本中启动内置后端
    this.backendProcess = spawn('./backend/lifetrace', [], {
      cwd: process.resourcesPath,
    })
  }
  
  private createMainWindow() {
    this.mainWindow = new BrowserWindow({
      width: 1200,
      height: 800,
      webPreferences: {
        preload: path.join(__dirname, 'preload.js'),
        contextIsolation: true,
        nodeIntegration: false,
      },
    })
    
    // 开发模式加载 localhost，生产模式加载打包文件
    if (isDev) {
      this.mainWindow.loadURL('http://localhost:3001')
    } else {
      this.mainWindow.loadFile('out/index.html')
    }
  }
}

## 开发规范

### 代码规范概览

| 层级 | 规范 | 工具 |
|------|------|------|
| **后端** | PEP 8 + 类型注解 | Ruff |
| **前端** | TypeScript 严格模式 | Biome |
| **API** | RESTful + OpenAPI | FastAPI 自动生成 |
| **Git** | Conventional Commits | pre-commit |

### 后端开发规范

详见：[BACKEND_GUIDELINES_CN.md](.github/BACKEND_GUIDELINES_CN.md)

关键点：
- 分层架构：Router → Service → Repository
- 类型注解：所有函数参数和返回值
- 文档字符串：Google 风格
- 错误处理：捕获特定异常
- 日志记录：使用 Loguru

### 前端开发规范

详见：[FRONTEND_GUIDELINES_CN.md](.github/FRONTEND_GUIDELINES_CN.md)

关键点：
- 组件命名：PascalCase
- 变量命名：camelCase
- 状态管理：服务端状态用 TanStack Query，客户端状态用 Zustand
- API 调用：使用 Orval 生成的 hooks
- 国际化：使用 next-intl，禁止硬编码文本

### Git 工作流

```bash
# 分支命名
feature/xxx    # 新功能
fix/xxx        # 修复
docs/xxx       # 文档
refactor/xxx   # 重构

# 提交信息格式 (Conventional Commits)
feat: 添加待办拖拽排序功能
fix: 修复日历视图日期显示错误
docs: 更新 API 文档
refactor: 重构 todo 服务层
```
