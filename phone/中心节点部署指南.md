# 中心节点部署指南（Phone）

本文用于 `phone` 连接你本机的 LifeTrace 中心节点，实现 Web 与 App 共用同一后端数据。

## 1. 一键启动（推荐）

在仓库根目录执行：

```powershell
cd E:\ffreeu\LifeTrace
scripts\start-center.bat
```

该脚本会启动：
- Phoenix
- AgentOS
- LifeTrace Backend
- Frontend
- cpolar 隧道（backend http / backend tcp / frontend http）

## 2. cpolar 手动命令（排障必备）

如果脚本没拉起 cpolar，或者你要单独调试，请手动执行：

### 2.1 查看 cpolar 是否可用

```powershell
& "C:\Program Files\cpolar\cpolar.exe" version
```

### 2.2 启动 backend HTTP 隧道

```powershell
& "C:\Program Files\cpolar\cpolar.exe" http 8002
```

带固定子域名（示例）：

```powershell
& "C:\Program Files\cpolar\cpolar.exe" http -region=cn_top -subdomain=your-backend-subdomain 8002
```

### 2.3 启动 frontend HTTP 隧道

```powershell
& "C:\Program Files\cpolar\cpolar.exe" http 3001
```

### 2.4 启动 backend TCP 隧道（命名隧道）

```powershell
& "C:\Program Files\cpolar\cpolar.exe" start backend_tcp
```

注意：`8001/8002` 以你后端实际监听端口为准。

## 3. 手机端配置后端 URL（App 内可改）

无需改代码文件，直接在 App 内配置：

1. 打开 App，进入开发者设置页
2. 找到 `Backend API Base URL`
3. 输入你的中心节点地址（示例：`https://你的-backend-cpolar-域名/`）
4. 点击保存
5. 关闭并重新打开 App（调试模式可 Hot Restart）

注意：
- URL 必须包含 `http://` 或 `https://`
- 建议保留末尾 `/`
- 若要恢复内置默认地址，在同一页面清空/重置自定义 URL 即可

## 4. 鉴权验证

```powershell
curl.exe -H "Authorization: Bearer lifetrace-omi-compat-2026" https://你的-backend-cpolar-域名/v1/users/me
```

返回用户 JSON（含 `uid`）表示鉴权正常。

## 5. App 内联通测试

中心节点测试页建议检查：
- `/v1/users/me`
- `/v1/conversations`
- `/v1/action-items`
- `/v2/messages` GET
- `/v2/messages` POST

## 6. 常见问题

### Q1: `Invalid or missing token`

- 检查 `Authorization: Bearer <token>` 格式
- 检查 token 是否与后端 `omi_compat.token` 一致

### Q2: 显示 8001，实际跑 8002

同时看两处：
- 后端日志：`Uvicorn running on http://0.0.0.0:xxxx`
- cpolar 日志：`Forwarding ... -> http://localhost:xxxx`

这两个端口必须一致。

### Q3: `cpolar` 不是内部或外部命令

直接用完整路径执行：

```powershell
& "C:\Program Files\cpolar\cpolar.exe" version
```

## 7. 推荐操作顺序

1. 启动中心节点：`scripts\start-center.bat`
2. 在 App 开发者设置的 `Backend API Base URL` 输入并保存 backend 公网 URL
3. 重启 App（或 Hot Restart）
4. 打开中心节点测试页验证接口
5. 再测试对话/待办/消息互通
