# 富交互通知卡片改造方案

> 基于 FreeTodo 现有的 `notification-popup-manager.ts` 进行改造，
> 从"3秒只读 toast"升级为"可交互的富通知卡片"，作为 Proactive Agent 主动服务的核心载体。

---

## 一、改造目标总览

| 维度 | 现状 | 改造后 |
| ---- | ---- | ------ |
| 可交互性 | 鼠标穿透，不可点击 | 可点击，支持操作按钮 |
| 内容丰富度 | title + message 两行文本 | 来源、内容、上下文、建议回复、操作按钮 |
| 自动消失 | 3秒固定 | 6秒，鼠标悬停时暂停倒计时 |
| 窗口尺寸 | 固定 360×120 | 根据卡片类型动态调整高度 |
| 按钮反馈 | 无 | 按钮点击通过 IPC 回传主进程，触发后续动作 |

## 二、卡片类型定义

### 四种标准交互模式

参考 Ambient Agent 的 Human-in-the-Loop 设计范式（LangChain Agent Inbox），通知卡片提供四种标准用户响应模式：

| 模式 | actionId | 说明 | 适用场景 |
|:---|:---|:---|:---|
| **采纳** | `accept` | 接受 AI 方案，直接执行 | AI 建议回复、自动创建待办等 |
| **编辑** | `edit` | 打开主窗口修改 AI 方案后再执行 | 需要调整回复措辞、修改待办内容等 |
| **稍后** | `defer` | 暂时搁置，加入托盘未处理列表 | 当前正忙，稍后回来处理 |
| **忽略** | `dismiss` | 跳过此通知，不做任何事 | 不需要处理的信息 |

不同卡片类型可选择性展示其中的子集（通过 `actions` 数组控制）。

### 卡片类型定义

不同场景触发的通知内容量不同，定义三种卡片类型，宽度统一 400px，高度自适应：

### Type A：轻量提醒（~140px 高）

最简单的通知，只有标题+消息+操作按钮。适用于简单待办提取等场景。

```
┌─────────────────────────────────────────┐
│  [icon]  待办提醒                        │
│                                         │
│  来源：微信 - 王强                       │
│  "明天下午3点打球，别忘了"               │
│                                         │
│  [ 采纳 ]  [ 稍后 ]  [ 忽略 ]            │
│  ▇▇▇▇▇▇▇▇▇░░░░  进度条                 │
└─────────────────────────────────────────┘
```

### Type B：标准通知（~220px 高）

带上下文信息，适用于文件查找、日程冲突等需要背景信息的场景。

```
┌─────────────────────────────────────────┐
```

### Type C：建议行动（~280px 高）

最丰富的卡片，带建议回复/建议操作，适用于需要 AI 给出行动建议的场景。

## 三、数据结构改造

### 3.1 PopupData 接口（改造前后对比）

**改造前**：

```typescript
interface PopupData {
  title?: string;
  message?: string;
}
```

**改造后**：

```typescript
/** 卡片类型 */
type CardType = "light" | "standard" | "action";

/** 操作按钮定义 */
interface PopupAction {
  id: string;          // 按钮唯一标识，回传给主进程
  label: string;       // 按钮显示文本
  style?: "primary" | "secondary" | "ghost";  // 按钮样式
}

/** 弹窗内容数据 */
interface PopupData {
  /** 通知唯一ID，用于追踪和去重 */
  id: string;
  /** 卡片类型，决定布局和窗口高度 */
  type: CardType;
  /** 来源标识（如 "微信 - 张教授"） */
  source: string;
  /** 主要内容（对方说了什么 / 发生了什么） */
  content: string;
  /** 相关上下文条目（可选，Type B/C 使用） */
  context?: string[];
  /** AI 建议的回复或操作描述（可选，Type C 使用） */
  suggestion?: string;
  /** 操作按钮列表 */
  actions: PopupAction[];
}
```

### 3.2 卡片类型与窗口尺寸映射

```typescript
const POPUP_WIDTH = 400;

const CARD_HEIGHT_MAP: Record<CardType, number> = {
  light:    160,   // Type A：轻量提醒
  standard: 240,   // Type B：标准通知
  action:   300,   // Type C：建议行动
```

## 四、核心改造点（逐项）

### 4.1 可点击：去掉鼠标穿透，但不抢焦点

**改造点**：`createWindow()` 方法

```typescript
// ❌ 删除这行
this.popupWindow.setIgnoreMouseEvents(true, { forward: true });

// ✅ 改为：窗口可接收鼠标事件，但不抢焦点
// focusable 保持 false，showInactive() 保持不变
// 这样用户可以点击按钮，但弹窗不会抢走当前应用的焦点
```

关键：`focusable: false` + `showInactive()` 的组合已经能保证不抢焦点。
去掉 `setIgnoreMouseEvents` 后，鼠标事件就能到达弹窗内的按钮了。

### 4.2 按钮点击回传：popup 窗口需要 preload

**问题**：当前 popup 窗口没有 preload 脚本，HTML 里的按钮点击无法通知主进程。

**方案**：为 popup 窗口添加专用的 preload 脚本。

新建 `electron/notification-popup-preload.ts`：

```typescript
import { contextBridge, ipcRenderer } from "electron";

contextBridge.exposeInMainWorld("popupAPI", {
  /** 用户点击了操作按钮 */
  onAction: (notificationId: string, actionId: string) => {
    ipcRenderer.send("notification-popup-action", { notificationId, actionId });
  },
  /** 鼠标进入弹窗区域（暂停倒计时） */
  onMouseEnter: () => {
    ipcRenderer.send("notification-popup-hover", { hovering: true });
  },
  /** 鼠标离开弹窗区域（恢复倒计时） */
  onMouseLeave: () => {
    ipcRenderer.send("notification-popup-hover", { hovering: false });
  },
});
```

`createWindow()` 中加入 preload：

```typescript
this.popupWindow = new BrowserWindow({
  // ...其他配置不变
  focusable: false,
  webPreferences: {
    nodeIntegration: false,
    contextIsolation: true,
    preload: path.join(__dirname, "notification-popup-preload.js"),
  },
});
```

### 4.3 鼠标悬停暂停倒计时

**逻辑**：鼠标进入弹窗时清除自动隐藏定时器，鼠标离开时重新开始倒计时（从剩余时间继续）。

在 `NotificationPopupManager` 中新增：

```typescript
private remainingMs: number = 0;
private showTimestamp: number = 0;
private isHovering: boolean = false;

/**
 * 注册悬停相关的 IPC 监听
 */
private registerHoverListeners(): void {
  ipcMain.on("notification-popup-hover", (_event, data: { hovering: boolean }) => {
    if (data.hovering) {
      // 鼠标进入：暂停倒计时
      this.isHovering = true;
      const elapsed = Date.now() - this.showTimestamp;
      this.remainingMs = Math.max(NOTIFICATION_DURATION_MS - elapsed, 1000);
      if (this.fadeTimeoutId) clearTimeout(this.fadeTimeoutId);
      if (this.hideTimeoutId) clearTimeout(this.hideTimeoutId);
      // 暂停进度条动画（通过 JS 注入）
      this.pauseProgressBar();
    } else {
      // 鼠标离开：用剩余时间恢复倒计时
      this.isHovering = false;
      this.showTimestamp = Date.now();
      this.scheduleHide(this.remainingMs);
      this.resumeProgressBar(this.remainingMs);
    }
  });
}
```

### 4.4 自动消失时间改为 6 秒

```typescript
/** 通知显示持续时间（毫秒）- 6 秒 */
const NOTIFICATION_DURATION_MS = 6_000;
```

### 4.5 动态窗口尺寸

**改造点**：`showNotification()` 方法中，根据 `data.type` 动态设置窗口大小和位置。

```typescript
private showNotification(data: PopupData): void {
  const height = CARD_HEIGHT_MAP[data.type] || CARD_HEIGHT_MAP.standard;
  const workArea = screen.getPrimaryDisplay().workArea;

  if (!this.popupWindow || this.popupWindow.isDestroyed()) {
    this.createWindow();
  }
  if (!this.popupWindow) return;

  // 动态调整窗口大小
  this.popupWindow.setSize(POPUP_WIDTH, height);
  // 重新定位到左下角（Y 坐标随高度变化）
  this.popupWindow.setPosition(
    workArea.x + MARGIN,
    workArea.y + workArea.height - height - MARGIN,
  );

  // 注入内容并显示...
}
```

## 五、按钮点击后的处理链路

```
用户点击按钮
    ↓
popup preload: popupAPI.onAction(notificationId, actionId)
    ↓
IPC → 主进程收到 "notification-popup-action"
    ↓
主进程处理：
  1. 立即关闭弹窗（播放退场动画后隐藏）
  2. 将 { notificationId, actionId } 转发给渲染进程
    ↓
IPC → 渲染进程收到动作事件
    ↓
渲染进程根据 actionId 执行对应逻辑（四种标准交互模式）：
  - "accept"  → 采纳：接受 AI 方案，直接执行（建议回复 / 加入待办等）
  - "edit"    → 编辑：打开主窗口，用户修改 AI 方案后再执行
  - "defer"   → 稍后：标记为稍后处理，加入托盘角标
  - "dismiss" → 忽略：不做任何事
```

在 `main.ts` 中注册：

```typescript
ipcMain.on("notification-popup-action", (_event, data) => {
  const { notificationId, actionId } = data;

  // 关闭弹窗
  notificationPopupManager?.dismiss();

  // 转发给渲染进程
  mainWindow?.webContents.send("notification-action", { notificationId, actionId });

  // 如果是 "edit"，同时打开主窗口进入编辑模式
  if (actionId === "edit") {
    mainWindow?.show();
    mainWindow?.focus();
  }
});
```
