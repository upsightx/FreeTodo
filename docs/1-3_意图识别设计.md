# 意图识别设计

> 本文档定义主动意图识别模块的设计方案。
> 意图识别的输入来自 [感知流](./1-1_感知流设计.md) 和 [Memory](./1-2_Memory模块设计.md) 提供的上下文。

---

## 一、意图来源分类

| 类型 | 描述 | 检测难度 | 优先级 |
|:---|:---|:---|:---|
| **社交传递型** | 别人通过某种媒介（屏幕消息、语音对话）把信息传递给你 | 较易 | P0 |
| **自发产生型** | 你脑子里自发产生的想法 | 较难 | P2 |

**先做社交传递型**：这类意图非常清晰，如"导师问能不能今天下午开个会"，一定是通过某种媒介到达用户感官的。

---

## 二、小模型初筛（When）

小模型负责实时、低成本地完成第一道筛选，判断是否需要触发大模型：

| 优先级 | 检测场景 | 检测方法 | 技术方案 |
|:---|:---|:---|:---|
| P0 | 收到疑问句 | 句式检测 | 小模型快速识别 |
| P0 | 收到祈使句 | 句式检测 | 小模型快速识别 |

**小模型职责**：检测疑问句/祈使句，低延迟低成本实时初筛，判断是否触发大模型。

---

## 三、大模型深度推理

```
小模型初筛通过 → 触发大模型
      ↓
┌─────────────────────────────────────────┐
│            大模型（强力）                 │
│                                          │
│  1. 进一步意图分析                        │
│     - 理解复杂意图、消歧                  │
│     - 判断用户真正需要什么                │
│                                          │
│  2. 内容检索（What）                      │
│     - 基于个人知识库的RAG检索             │
│     - 跨时间线信息关联                    │
│     - 联想：用户没想到的内容AI能想到       │
│                                          │
│  3. 规划与响应（How）                     │
│     - 建议回复（帮用户想回复话术）         │
│     - 尝试回答（先帮用户回答问题）         │
│     - 自动执行（指派Sub-Agent完成任务）    │
└─────────────────┬───────────────────────┘
                  ↓ 需要执行任务时
┌─────────────────────────────────────────┐
│           Sub-Agent调度                  │
│  任务分类 → 选择Agent → 执行 → 推送结果 │
└─────────────────────────────────────────┘
```

**大模型职责**：意图分析与消歧，检索 Memory（RAG），规划响应 + 调度 Sub-Agent。
