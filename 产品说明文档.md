## 产品概述

### 什么是 FreeTodo？

**FreeTodo** 是由 FreeU Group（为您分忧）打造的一款 AI 驱动的智能待办管理应用。它不仅仅是一个简单的 To-Do List，而是一个帮助用户实现**认知卸载**的智能助手，通过对话式 AI 交互和智能任务拆分，将复杂项目转化为可执行的行动步骤。

### 产品特色

| 特色 | 描述 |
|------|------|
| 🤖 **AI 智能助手** | 智能任务拆分、上下文感知建议、智能消息提取 |
| ✅ **全面任务管理** | 层级任务结构、四级优先级、标签分类、截止日期管理 |
| 📅 **多视图日历** | 日/周/月视图、拖拽排期、快速创建待办 |
| 🎨 **现代化界面** | 多面板布局、深色/浅色主题、国际化支持 |
| 💻 **跨平台支持** | Web 应用 + Electron 桌面应用（Windows/macOS） |

### 核心理念

FreeTodo 有且只有一个核心功能——**认知卸载**

> "Free Todo" 的含义：**放手去做，放开手去干**

通过 AI 辅助和智能规划：
- 减轻用户的输入和处理负担
- 从"混沌"到"秩序"，将无序的输入转化为有序的任务
- 提供心理安全感和温暖可靠的秘书伙伴体验

## 核心功能

### 🤖 AI 智能助手

#### 智能任务拆分
- **功能描述**：AI 自动将复杂任务分解为可管理的子任务
- **交互方式**：通过引导式问卷流程完成任务拆分
- **使用场景**：面对"完成年度报告"这样的大任务，AI 会引导拆分为"收集数据"、"分析数据"、"撰写报告"等具体步骤

#### 智能任务提取
- **功能描述**：从 AI 对话响应中自动提取可执行的待办事项
- **工作原理**：分析对话内容，识别行动项，自动创建待办
- **使用场景**：与 AI 讨论项目计划后，自动生成对应的待办列表

#### 上下文感知建议
- **功能描述**：AI 根据当前待办上下文提供任务建议
- **智能程度**：理解任务之间的关联，给出合理的优先级和时间建议
- **使用场景**：基于已有待办，AI 主动提醒可能遗漏的相关任务

### ✅ 全面的任务管理

#### 层级任务结构
```
项目级任务
├── 子任务 1
│   ├── 子子任务 1.1
│   └── 子子任务 1.2
├── 子任务 2
└── 子任务 3
```
- 支持父子任务关系，无限层级嵌套
- 父任务完成度自动根据子任务计算

#### 优先级与状态管理

| 优先级 | 颜色标识 | 使用场景 |
|--------|----------|----------|
| 🔴 紧急 | 红色 | 需要立即处理的事项 |
| 🟠 高 | 橙色 | 重要但不紧急 |
| 🟡 中 | 黄色 | 常规任务 |
| 🟢 低 | 绿色 | 可延后处理 |

任务状态：待处理 → 进行中 → 已完成 / 已取消

#### 标签与分类
- 自定义标签创建和管理
- 按标签快速筛选任务
- 支持多标签关联

#### 截止日期管理
- 日期选择器设置截止日期
- 可视化提醒（即将到期、已过期）
- 与日历视图联动

### 📅 多视图日历

#### 三种视图模式
- **日视图**：精细到小时的时间管理
- **周视图**：一周任务总览
- **月视图**：宏观月度规划

#### 拖拽排期
- 直接拖拽待办到日历时间槽
- 自动更新任务的计划时间
- 支持跨天拖拽调整

#### 快速创建
- 点击日历空白区域快速创建待办
- 自动填充选中的日期和时间

### 🎨 现代化用户界面

#### 多面板布局

## 使用场景

### 场景一：项目规划

**背景**：产品经理李明需要规划一个新产品的上线计划

**使用流程**：
1. 在 AI 聊天面板输入："帮我规划新产品上线的任务清单"
2. AI 通过问卷式交互了解产品类型、时间节点等信息
3. AI 自动生成结构化的任务列表
4. 李明在待办列表中调整优先级和截止日期
5. 通过日历视图查看整体时间安排

### 场景二：会议跟进

**背景**：开完周会后需要整理和跟进会议待办

**使用流程**：
1. 打开 AI 聊天面板
2. 粘贴会议记录或语音转文字内容
3. 输入："帮我从这些内容中提取待办事项"
4. AI 自动识别并创建相关待办
5. 为每个待办设置截止日期和负责人标签

### 场景三：日常任务管理

**背景**：每天管理工作和生活中的各种事项

**使用流程**：
1. 早晨打开应用查看今日待办（通过日历日视图）
2. 拖拽调整今日任务优先级
3. 完成任务后打勾标记
4. 对于复杂任务，与 AI 对话进行拆分
5. 晚上回顾今日完成情况

### 场景四：长期目标管理

**背景**：管理年度目标和季度 OKR

**使用流程**：
1. 创建年度目标作为父任务
2. 拆分为季度目标作为子任务
3. 进一步拆分为月度/周度可执行任务
4. 使用月视图查看整体进度
5. 定期与 AI 回顾和调整计划

## 快速入门

### 环境要求

**后端**：
- Python 3.12
- Windows / macOS

**前端**：
- Node.js 20+
- pnpm 包管理器

### 安装步骤

#### 1. 克隆项目
```bash
git clone https://github.com/FreeU-group/LifeTrace.git
cd LifeTrace
```

#### 2. 安装后端依赖
```bash
# 安装 uv 包管理器
# Windows
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# 同步依赖
uv sync

# 激活虚拟环境
.venv\Scripts\activate  # Windows
source .venv/bin/activate  # macOS/Linux
```

#### 3. 启动后端服务
```bash
python -m lifetrace.server
```

#### 4. 安装并启动前端
```bash
cd free-todo-frontend
pnpm install
pnpm dev
```

#### 5. 访问应用
打开浏览器访问 `http://localhost:3001`

### 配置说明

**后端配置文件**：`lifetrace/config/config.yaml`
- LLM API 配置
- 服务端口设置
- 数据库路径配置

**AI 提示词配置**：`lifetrace/config/prompt.yaml`
- 自定义 AI 对话提示词
- 任务拆分提示词配置

## 常见问题

### Q1：FreeTodo 和其他待办应用有什么区别？

**A**：FreeTodo 的核心差异在于 AI 集成深度：
- 不只是添加 AI 聊天，而是将 AI 深度融入任务管理流程
- 智能任务拆分帮助处理复杂项目
- 上下文感知提供个性化建议
- 与 LifeTrace 生态协同，未来支持主动服务

### Q2：数据存储在哪里？是否安全？

**A**：
- 所有数据默认存储在本地（SQLite 数据库）
- 不上传用户数据到云端
- LLM 调用仅发送必要的对话内容
- 未来计划支持完全本地的大模型

### Q3：是否支持团队协作？

**A**：
- 当前版本专注于个人使用
- 团队协作功能在未来路线图中
- 可通过标签和分类管理不同项目

### Q4：如何配置 AI 功能？

**A**：
1. 编辑 `lifetrace/config/config.yaml`
2. 配置 LLM 提供商和 API Key
3. 支持 OpenAI、通义千问等多种模型
4. 重启后端服务生效
